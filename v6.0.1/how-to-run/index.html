How To Run

Pipelines should be run WITH A SINGLE SAMPLE AT TIME. Otherwise resource allocation and Nextflow errors could cause the pipeline to fail

  1. Make sure the pipeline is already downloaded to yoru machine. You can either download the stable release or the dev version by cloning the repo.

  2. Create a config file for input, output, and parameters. An example can be found here. See Inputs for description of each variables in the config file.

  3. Update the input csv. See Inputs for the columns needed. All columns must exist in order to run the pipeline. An example can be found here. The example csv is a single-lane sample, however this pipeline can take multi-lane sample as well, with each record in the csv file representing a lane (a paire of fastq). All records must have the same value in the sample column.

  4. See the submission script, here, to submit your pipeline

BWA-MEM2 Genome Index The genome index files must be generated by BWA-MEM2 with the correct version. Neither the index from previous version of BWA-MEM2, or from BWA can be accepted. The index files can be generated from a custom genome FASTA using the command below:

docker run \
    -it --rm \
    -v $(pwd):$(pwd) \
    -w $(pwd) \
    blcdsdockerregistry/align-dna:bwa-mem2-2.1_samtools-1.10 \
    bwa-mem2 index genome.fa
